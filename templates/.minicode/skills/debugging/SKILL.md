# 调试技能

当用户报告 Bug 或需要排查问题时，严格按本技能的系统化流程执行。

## 调试流程

### 1. 理解问题
- **现象**：什么行为是不正确的？
- **期望**：正确的行为应该是什么？
- **复现**：如何稳定复现？（步骤、环境、数据）
- **范围**：是偶发还是必现？什么时候开始的？

### 2. 收集信息
- 用 `grep` 搜索相关代码，定位涉及的模块
- 用 `read_file` 阅读相关代码，理解逻辑
- 用 `bash` 查看日志、错误堆栈
- 检查最近的 Git 变更：`git log --oneline -10` 和 `git diff HEAD~3`

### 3. 形成假设
基于收集的信息，列出可能的原因（假设），按可能性排序：
1. 假设 A — （可能性 80%）...
2. 假设 B — （可能性 15%）...
3. 假设 C — （可能性 5%）...

### 4. 验证假设
从最可能的假设开始验证：
- **添加日志**：在关键位置添加 `console.log` 或日志输出
- **断点调试**：用 `debugger_page` 工具（前端）或 Node.js Inspector
- **二分法**：注释掉部分代码，逐步缩小范围
- **最小复现**：提取最小可复现的代码片段

### 5. 定位根因
确认假设成立后，定位到具体的代码行，分析为什么会产生这个 Bug。

### 6. 修复
- 提出最小改动的修复方案
- 用 `confirm` 确认后执行修复
- 修复时注意不要引入新问题

### 7. 验证
- 运行相关测试
- 按用户的复现步骤验证问题已解决
- 检查是否有类似的代码也存在同样问题

### 8. 预防
- 建议添加测试防止回归
- 建议添加类型检查或输入校验
- 建议添加日志方便未来排查

## 常见 Bug 类型速查

| 类型         | 典型表现                           | 排查方向                         |
| ------------ | ---------------------------------- | -------------------------------- |
| 空值错误     | `Cannot read property of null/undefined` | 检查数据来源、异步时序    |
| 类型错误     | `TypeError: x is not a function`   | 检查 import/export、this 绑定    |
| 异步问题     | 数据时有时无、顺序不确定           | 检查是否漏了 await、Promise 链   |
| 闭包陷阱     | 循环中获取的变量值都是最后一个     | 用 let 替代 var、使用 IIFE       |
| 竞态条件     | 偶发性错误、刷新后正常             | 检查异步操作共享状态             |
| 路径问题     | 文件找不到、路径拼接错误           | 检查路径分隔符、相对/绝对路径    |
| 编码问题     | 乱码、特殊字符异常                 | 检查 UTF-8 BOM、编码转换         |
| 内存泄漏     | 内存持续增长、页面越来越卡         | 检查事件监听、定时器、闭包引用   |
| CSS 层叠     | 样式不生效或被覆盖                 | 检查优先级、 specificity         |

## 输出格式

```
## Bug 排查报告

### 问题描述
（现象 + 期望 + 复现步骤）

### 根因分析
（定位到文件:行号，解释原因）

### 修复方案
（最小改动的修复代码）

### 验证结果
（如何确认修复有效）

### 预防建议
（如何避免类似问题）
```
