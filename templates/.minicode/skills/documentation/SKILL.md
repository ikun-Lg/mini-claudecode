# 文档编写技能

当用户要求编写文档时，根据文档类型遵循对应规范。

## 文档类型

### 1. README.md
项目入口文档，应包含以下章节：

```markdown
# 项目名称

> 一句话描述项目是什么

## 功能特性
- 核心功能 1
- 核心功能 2

## 快速开始
### 环境要求
- Node.js >= 18
### 安装
（安装命令）
### 配置
（配置说明）
### 运行
（运行命令）

## 使用方式
（基本用法 + 示例）

## 项目结构
（目录树 + 说明）

## 技术栈
| 技术 | 用途 |
| --- | --- |

## 贡献指南
（如何参与开发）

## License
```

**原则**：
- 开头一句话说清项目是什么
- 快速开始部分确保用户 5 分钟内能跑起来
- 代码示例必须可复制粘贴运行

### 2. API 文档
接口文档应包含：

```markdown
## 接口名称
一句话描述接口作用

### 请求
- 方法：GET/POST/...
- 路径：/api/xxx
- 参数：

| 参数名 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |

### 响应
```json
{
  "code": 200,
  "data": {}
}
```

### 示例
（请求 + 响应的完整示例）
```

### 3. 代码注释

#### JSDoc 格式
```javascript
/**
 * 函数描述（一句话说明作用）
 * @param {Type} name - 参数说明
 * @returns {Type} 返回值说明
 * @throws {Error} 异常说明
 */
```

#### 注释原则
- 注释解释「为什么」，不解释「是什么」（代码本身应能表达是什么）
- 复杂逻辑必须有注释
- TODO 格式：`// TODO(作者): 描述`
- 不保留被注释掉的代码（用 Git 管理历史）

### 4. 变更日志 (CHANGELOG.md)
```markdown
# Changelog

## [1.1.0] - 2024-01-15
### Added
- 新增 xxx 功能
### Changed
- 修改了 xxx
### Fixed
- 修复了 xxx
### Removed
- 移除了 xxx
```

## 编写原则

1. **受众意识**：明确文档的读者是谁（开发者/用户/运维），调整语言深度
2. **示例驱动**：每个功能点都配可运行的示例
3. **保持更新**：代码变更时同步更新文档
4. **简洁明了**：避免冗长描述，用表格和列表提升可读性
5. **结构清晰**：用标题层级组织内容，方便快速定位
