# 性能优化

请帮我分析并优化指定代码的性能。优化时遵循以下原则：

## 优化原则

1. **先测量后优化**：不要凭感觉优化，先用数据定位瓶颈
2. **聚焦热点**：优先优化耗时占比最大的部分（80/20 原则）
3. **空间换时间 / 时间换空间**：根据场景权衡，不要盲目追求某一方向
4. **不牺牲可读性**：优化后的代码仍需可维护，复杂优化需添加注释说明

## 常见性能问题

### 前端
| 问题             | 检测方式                   | 优化方案                          |
| ---------------- | -------------------------- | --------------------------------- |
| 不必要的重渲染   | React DevTools Profiler    | `memo`、`useMemo`、`useCallback`  |
| 大列表渲染卡顿   | Performance 面板           | 虚拟列表（react-window 等）       |
| 打包体积过大     | webpack-bundle-analyzer    | 代码分割、Tree Shaking、懒加载    |
| 图片加载慢       | Network 面板               | 懒加载、WebP、CDN、压缩           |
| 首屏白屏         | Lighthouse                 | SSR/SSG、预加载、骨架屏           |

### Node.js
| 问题             | 检测方式                   | 优化方案                          |
| ---------------- | -------------------------- | --------------------------------- |
| 同步阻塞         | `console.time` / Profiler  | 改为异步 I/O                      |
| 内存泄漏         | `--inspect` 堆快照         | 释放引用、避免全局变量累积        |
| 数据库 N+1 查询  | 查询日志                   | 批量查询、JOIN、预加载            |
| JSON 序列化慢    | `JSON.parse` 计时          | 流式解析、换用 faster-json        |
| 大文件处理       | 内存监控                   | 流式处理（Stream）                |

## 执行步骤

1. 用 `read_file` 阅读目标代码，理解功能
2. 分析潜在的性能瓶颈
3. 如果可以，用 `bash` 工具运行性能测试或 Profiler
4. 提出优化方案，说明预期收益
5. 用 `confirm` 工具确认后执行优化
6. 优化后再次测量，对比前后数据

## 输出格式

```
## 性能分析

### 当前瓶颈
（定位到的性能热点及数据）

### 优化方案
（具体的优化措施及预期收益）

### 优化后对比
（优化前后的性能数据对比）

### 进一步建议
（后续可继续优化的方向）
```
