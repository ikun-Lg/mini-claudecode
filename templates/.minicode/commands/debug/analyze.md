# Bug 分析与调试

请帮我分析和定位 Bug。分析时遵循以下步骤：

## 调试步骤

### 1. 收集信息
- 仔细阅读用户描述的问题现象、复现步骤和期望行为
- 用 `grep` 搜索相关代码，定位可能出问题的模块
- 用 `read_file` 阅读相关代码，理解逻辑

### 2. 分析根因
- 对比「期望行为」与「实际行为」，找出差异点
- 检查常见 Bug 来源：
  - 空值/未定义值未处理
  - 异步操作未正确 await
  - 数据类型隐式转换
  - 作用域问题（变量提升、闭包陷阱）
  - 竞态条件
  - 边界条件遗漏（off-by-one、空数组等）
  - 环境差异（路径分隔符、换行符、编码）

### 3. 提出修复方案
- 给出根因分析说明
- 提供修复方案（优先最小改动，避免引入新问题）
- 用 `confirm` 工具确认后执行修复

### 4. 验证修复
- 如果可以运行测试，执行相关测试验证
- 如果项目有调试页面，可用 `debugger_page` 工具截图验证
- 建议用户复现问题确认修复

## 输出格式

```
## Bug 分析

### 问题现象
（用户描述的问题）

### 根因分析
（定位到具体文件和行号，解释为什么会出问题）

### 修复方案
（最小改动的修复代码）

### 验证方式
（如何确认修复有效）

### 预防建议
（如何避免类似问题，如添加类型检查、边界处理等）
```
