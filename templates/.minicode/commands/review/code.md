# 代码审查

请对以下代码进行全面的代码审查。审查时请遵循以下检查清单：

## 审查清单

### 1. 功能正确性
- 代码是否正确实现了预期功能
- 边界条件是否处理完备（空值、零值、越界、并发等）
- 错误处理是否完善（异常捕获、错误传播、降级策略）

### 2. 代码质量
- 命名是否清晰且有意义（变量、函数、类名）
- 函数是否职责单一，长度是否合理（建议不超过 50 行）
- 是否有重复代码可以抽取
- 注释是否充分且有价值（解释「为什么」而非「做了什么」）

### 3. 安全性
- 是否存在注入风险（SQL 注入、XSS、命令注入）
- 敏感信息是否妥善处理（密码、Token、密钥不硬编码）
- 输入校验是否充分
- 权限控制是否正确

### 4. 性能
- 是否有明显的性能问题（N+1 查询、不必要的循环、内存泄漏）
- 数据结构选择是否合理
- 是否可以利用缓存或懒加载优化

### 5. 可维护性
- 代码结构是否清晰，易于理解和修改
- 是否遵循项目现有的代码风格和约定
- 模块间耦合度是否合理

## 输出格式

请按以下格式输出审查结果：

```
## 审查总结
（一句话总结代码整体质量）

## 问题列表
### 🔴 严重问题（必须修复）
- [文件:行号] 问题描述 + 修复建议

### 🟡 建议改进（推荐修复）
- [文件:行号] 问题描述 + 改进建议

### 🟢 小问题（可选修复）
- [文件:行号] 问题描述

## 优点
- 值得肯定的设计或实现
```
